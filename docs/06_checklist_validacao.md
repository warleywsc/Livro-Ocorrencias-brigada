# Checklist de Validação

## 1. Identidade do projeto

- [ ] Nome oficial usado corretamente.
- [ ] Interface em português do Brasil.
- [ ] Nenhum termo proibido foi usado.
- [ ] Visual aprovado foi respeitado como referência.

## 2. Shell global

- [ ] Header é componente único.
- [ ] Sidebar é componente único.
- [ ] Footer é componente único.
- [ ] Footer está no fluxo normal da página.
- [ ] Footer não está fixed, sticky, absolute ou sobreposto.
- [ ] Botões finais ficam antes do footer.

## 3. Navegação

- [ ] Menu lateral contém somente os 8 itens oficiais.
- [ ] Dashboard acessível.
- [ ] Livro do Plantão acessível.
- [ ] Ocorrências acessível.
- [ ] Pendências acessível.
- [ ] POTs e Ciência acessível.
- [ ] Histórico acessível.
- [ ] Relatórios acessível.
- [ ] Administração acessível conforme permissão.

## 4. Dados

- [ ] Lista Usuarios validada.
- [ ] Lista Plantoes validada.
- [ ] Lista EquipePlantao validada.
- [ ] Lista ViaturasMateriais validada.
- [ ] Lista ConferenciasViaturasMateriais validada.
- [ ] Lista InspecoesRotina validada.
- [ ] Lista Ocorrencias validada.
- [ ] Lista PendenciasOperacionais validada.
- [ ] Lista POTs validada.
- [ ] Lista CienciaPOT validada.
- [ ] Lista ParametrosSistema validada.
- [ ] Lista Auditoria validada.

## 5. Permissões

- [ ] Perfil Leitura validado.
- [ ] Perfil Operador validado.
- [ ] Perfil Líder validado.
- [ ] Perfil Supervisor validado.
- [ ] Perfil Administrador validado.
- [ ] Livro finalizado bloqueado para edição comum.
- [ ] Reabertura restrita.
- [ ] Auditoria de ações críticas registrada.

## 6. Fluxo operacional

- [ ] Abrir plantão.
- [ ] Registrar equipe.
- [ ] Conferir viaturas e materiais.
- [ ] Registrar inspeção de rotina.
- [ ] Registrar ocorrência.
- [ ] Criar pendência.
- [ ] Acompanhar pendência.
- [ ] Registrar ciência em POT.
- [ ] Finalizar livro.
- [ ] Consultar histórico.
- [ ] Gerar relatório básico.
